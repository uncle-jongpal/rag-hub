# V4 평가 인사이트 - 한국어 위키 100문서

V4 단계에서 한국어 위키 98문서 (다양한 도메인) + 질문 30개로 13개 RAG 기법을 재평가했습니다. V3 데모셋(30문서/12질문) 대비 약 10배 큰 데이터셋입니다.

## 1. 평가 셋업

1. 문서 - 한국어 위키 98개 (다양한 도메인 - 기술 / 과학 / 한국 문화·역사 / 사회·경제 / 예술·스포츠 / 일반)
2. 총 글자 수 - 약 221,000자, 평균 2,261자, 최단 117자 / 최장 6,000자
3. 질문 30개 - 사실 16, 추론/비교 8, 요약/조망 7 (일부 태그 중복)
4. 생성 LLM - mistral-small-latest
5. 평가 LLM - mistral-large-latest (RAGAS)
6. 임베딩 - BGE-M3 (자체 답변 적합성 계산도 동일)

## 2. 완료된 7개 기법 평균 점수

순서는 충실도 / 답변 적합성 / 검색 정확도 / 검색 완전성

1. 01-naive : 0.824 / 0.710 / 0.739 / 0.767
2. 02-hybrid-search : 0.808 / 0.717 / 0.699 / 0.767
3. 03-reranking : 0.804 / 0.698 / 0.724 / 0.733
4. 04-hyde : 0.783 / 0.696 / 0.833 / 0.806
5. 05-multi-query : 0.832 / 0.724 / 0.708 / 0.783
6. 11-raptor : 0.833 / 0.725 / 0.762 / 0.761
7. 13-adaptive-rag : 0.698 / 0.689 / 0.687 / 0.800 (limit 20)

## 3. 누락된 6개 기법

미스트랄 무료 티어 분당 + 일일 한도가 평가 시작 시점에 닫혀 있어 즉시 실패한 기법들입니다.

1. 06-parent-child
2. 07-contextual-retrieval (인덱싱 비용 큰 기법)
3. 08-self-rag
4. 09-crag
5. 10-graphrag (인덱싱 비용 큰 기법)
6. 12-agentic-rag

한도 회복 후 재평가 일정 잡으면 같은 데이터셋으로 점수 채울 수 있습니다.

## 4. V3 와 다른 결정적 차이

V3 데모셋(30문서/12질문) 에서는 거의 모든 기법이 검색 완전성 1.000 으로 천장을 쳤습니다. 결과적으로 기법 간 차이가 잘 안 보였습니다.

V4 위키 데이터셋에서는 모든 메트릭이 분산됩니다.

1. 충실도 - 0.698 ~ 0.833
2. 답변 적합성 - 0.689 ~ 0.725
3. 검색 정확도 - 0.687 ~ 0.833
4. 검색 완전성 - 0.733 ~ 0.806

즉 큰 데이터셋이 기법 간 진짜 차이를 드러냅니다.

## 5. 메트릭별 1위와 의미

1. 충실도 1위 - 11-raptor (0.833), 05-multi-query (0.832 거의 동률)
   1) 트리 인덱스의 상위 레벨 요약과 다중 질문 변형이 LLM 에 안정적 컨텍스트를 공급해 환각이 줄어듬
2. 답변 적합성 1위 - 11-raptor (0.725), 05-multi-query (0.724)
   1) 위와 같은 이유. 균형 잡힌 컨텍스트가 답을 질문 의도에 잘 맞춤
3. 검색 정확도 1위 - 04-hyde (0.833)
   1) 가설 답변 임베딩이 위키 본문과 의미 매칭이 뛰어남. 답변 어휘 분포로 검색하니 무관 청크 적게 들어옴
4. 검색 완전성 1위 - 04-hyde (0.806)
   1) 같은 이유. HyDE 가 검색 두 메트릭 모두에서 압도적 1위

## 6. 도메인 특성 별 발견

위키처럼 본문이 풍부하고 정형화된 사실 기반 데이터에서

1. 질문/답변 처리 기법(HyDE, Multi-query) 이 검색 강화 기법(Hybrid, Reranking) 보다 효과가 더 큽니다
2. BM25 어휘 매칭(Hybrid) 이 의외로 정확도를 떨어뜨립니다 (V3 에서는 강점이었음). 위키처럼 어휘 일치가 의미 일치와 어긋날 가능성이 높은 도메인 영향
3. 트리 인덱싱(RAPTOR) 이 안정적인 균형 점수 - 다양한 질문 유형에 대응
4. 적응형 분기(Adaptive RAG) 가 의외 약점 - 한국어 위키 질문에 분류기가 약함

## 7. 운영 관점 시사점

1. 한국어 위키/사내 문서/매뉴얼처럼 사실 풍부한 도메인에는 HyDE 가 검색 정밀도/회수율 모두 1순위
2. 환각 통제가 가장 중요한 답변 도메인에는 RAPTOR 또는 Multi-query
3. 두 가지를 결합 (HyDE + Reranking 또는 RAPTOR + Multi-query) 하면 메트릭 두 축 모두 잡을 가능성
4. 단순 임베딩 검색 (Naive) 도 충실도 0.824 로 의외로 강력. 빠른 PoC 시작점으로 여전히 유효

2. 미스트랄 무료 티어 한도가 V4 같은 큰 데이터셋 평가에 부족 - 사내 LLM 또는 유료 API 전환 검토
