# V4 평가 인사이트 V2 - Qwen3.6 추가 측정

V4 첫 평가에서 미스트랄 무료 티어 한도로 누락된 6개 기법을 사내 Qwen3.6-27B 로 재측정해서 V4 13개 기법을 모두 완성했습니다. 이 문서는 V1 인사이트 문서를 보강한 V2 버전입니다.

## 1. 두 가지 평가 LLM 운영 배경

V4 평가 LLM 이 두 가지로 나뉜 이유

1. 미스트랄 7개 - 미스트랄 무료 티어가 처음 평가에서 살아 있던 동안 측정. mistral-small-latest (생성) + mistral-large-latest (평가)
2. Qwen 6개 - 미스트랄 한도가 닫혀서 사내 Qwen3.6-27B (생성 + 평가 동일) 로 재측정. 결과만 얻기 위한 임시 어댑터로 진행

평가 LLM 차이로 인한 self-judging bias 가능성

1. RAGAS 는 LLM-as-judge 방식이라 평가 LLM 의 자기 답변 우호 경향이 점수에 들어갈 수 있습니다
2. 미스트랄 평가는 생성 small + 평가 large 로 비대칭이라 self-judging bias 약함
3. Qwen 평가는 생성과 평가가 같은 모델이라 self-judging bias 강할 가능성. Qwen 점수가 미스트랄 대비 전반적으로 약간 후하게 나옴
4. 절대값 직접 비교는 조심하고, 같은 그룹 (미스트랄 7개 내, Qwen 6개 내) 안에서의 상대 비교가 신뢰도 더 높습니다

## 3. Qwen 측정에서의 핵심 발견

### 3-1. Self-RAG 의 분류기가 강한 모델에 휘둘림

Self-RAG 의 "검색 필요 여부 판정" 단계에서 Qwen3.6-27B 가 모든 질문에 대해 "내가 직접 답할 수 있음" 으로 판정했습니다. 결과적으로 검색을 한 번도 안 했고 검색 메트릭 3개가 모두 0 이 됐습니다 (컨텍스트가 빈 list).

다만 답변 적합성 0.779 는 다른 기법 대비 오히려 높음. Qwen 의 일반 지식이 위키 질문에 충분히 답할 수 있을 만큼 강해서 검색 안 해도 답이 좋다는 신호. 환각 위험은 별도 검증 필요합니다.

### 3-2. GraphRAG 가 위키 도메인에서 큰 약점

98 문서 → 520 청크 → 887 커뮤니티 라는 비정상적으로 많은 커뮤니티가 생겼습니다. 위키처럼 도메인이 다양한 데이터 (인공지능부터 한국 역사 / 음악 / 스포츠) 에서는 엔티티 그래프가 작은 컴포넌트 여러 개로 파편화되어 그래프 인덱싱의 강점이 사라집니다.

운영 시사점 - GraphRAG 는 도메인 응집적 데이터 (한 회사 문서 / 한 도메인 논문) 에서만 가치 있고, 위키처럼 다양한 도메인 흩어진 자료에는 적합하지 않습니다.

### 3-3. CRAG 와 Contextual Retrieval 가 검색 메트릭 동률 강자

1. 09-crag : 검색 정확도 0.960 / 검색 완전성 1.000
2. 07-contextual-retrieval : 검색 정확도 0.963 / 검색 완전성 1.000

두 기법 모두 위키 같은 사실 풍부한 데이터에서 검색 두 메트릭 거의 완벽. CRAG 가 인덱싱 비용 훨씬 작은데 비슷한 점수라 운영 효율 측면에서 CRAG 가 더 좋은 선택일 수 있습니다.

### 3-4. Parent-Child 가 충실도 1.000 으로 완벽

작은 청크로 검색하고 큰 부모 청크로 LLM 컨텍스트 보강하는 구조가 Qwen3.6 의 충실한 답변 생성과 결합되어 충실도 1.000. 위키처럼 본문 풍부한 도메인에서 부모 청크가 LLM 에 충분한 맥락을 제공하니까 환각이 거의 없습니다.

### 3-5. Agentic RAG 답변 적합성 약점

답변 적합성 0.523 으로 Qwen 측정 6개 중 가장 낮음. 미스트랄 측정 (0.723) 과도 같은 패턴. ReAct 다단계 도구 호출 거치면서 답이 원 질문 임베딩과 멀어지는 본질적 약점이 평가 LLM 무관하게 일관됩니다.

## 4. 미스트랄 측정에서의 핵심 발견 (V1 문서 요약)

V1 인사이트 문서의 핵심 발견 재확인

1. HyDE 가 검색 메트릭 압도적 1위 (정확도 0.833 / 완전성 0.806)
2. RAPTOR 와 Multi-query 가 답변 메트릭 동률 1위 (충실도 약 0.83 / 답변 적합성 약 0.72)
3. 위키 도메인에서 BM25 어휘 매칭 (Hybrid) 이 V3 와 반대로 약점

## 5. 통합 결론 - 위키/사실 풍부 도메인의 RAG 기법 선호도

종합하면 이 도메인에는 다음 순서대로 적합

1. 가장 강력 - HyDE (검색 메트릭 최고) 또는 Contextual Retrieval / CRAG (검색 + 충실도 균형)
2. 그 다음 - Parent-Child Chunking (충실도 최강) 또는 RAPTOR (균형)
3. 운영 효율 - CRAG (Contextual Retrieval 와 점수 비슷한데 인덱싱 비용 작음)
4. 신중 - Self-RAG (강한 모델에서 검색 스킵), Agentic (답변 적합성 약함)
5. 비추 - GraphRAG (도메인 파편화 시 무력)

## 6. 한계와 다음 단계

1. 평가 LLM 두 가지 운영으로 절대값 직접 비교가 약간 흐려졌습니다. 다음 평가는 한 모델로 통일 필요 (Qwen3.6 한 모델로 전부)
2. limit 도 일부 30, 일부 20, 일부 15 로 갈렸습니다. 같은 limit 로 통일 재측정 권장
3. Qwen self-judging bias 영향 추정 - 미스트랄 7개와 절대값 차이 약 0.1-0.15 정도 후하게 나옴
4. 위키 외 도메인 (법률 / 의료 / 사내 매뉴얼) 으로 V5 확장하면 GraphRAG 등 일부 기법의 진가 검증 가능
