# 기법 비교

7개 기법의 특성을 한 곳에서 비교합니다. 디스코드/문서 룰에 따라 마크다운 표 대신 줄바꿈 + 콜론 형식으로 정리합니다.

## 1. 01-naive

용도 : 베이스라인
인덱싱 비용 : 낮음 (임베딩만)
검색 시점 추가 호출 : 없음
강점 : 단순, 빠름
약점 : 어휘 매칭 약함, 청크 단독 모호성
구현 난이도 : ★☆☆☆☆

## 2. 02-hybrid-search

용도 : 어휘 + 의미 결합
인덱싱 비용 : 중간 (BM25 + Dense)
검색 시점 추가 호출 : BM25 점수 계산 (CPU)
강점 : 고유명사/약어, 한국어 어휘 매칭
약점 : 운영 복잡도 증가, RRF k 튜닝
구현 난이도 : ★★☆☆☆

## 3. 03-reranking

용도 : 검색 정밀도 향상
인덱싱 비용 : 낮음
검색 시점 추가 호출 : Cross-encoder N쌍 (GPU 권장)
강점 : 정밀도 큰 향상, Hybrid와 직교
약점 : 1차 검색 recall에 의존, GPU 필요
구현 난이도 : ★★☆☆☆

## 4. 04-hyde

용도 : 질문/답변 어휘 차이 보완
인덱싱 비용 : 낮음
검색 시점 추가 호출 : LLM 1회 (가설 답변)
강점 : 짧은 질문에 효과, 구현 간단
약점 : 지연 + 비용 증가, 가설이 빗나가면 역효과
구현 난이도 : ★★☆☆☆

## 5. 05-multi-query

용도 : 회수율(Recall) 향상
인덱싱 비용 : 낮음
검색 시점 추가 호출 : LLM 1회 + 검색 N배
강점 : 모호한 질문에 강함
약점 : 검색 호출 N배, 변형 질문 품질이 LLM 의존
구현 난이도 : ★★☆☆☆

## 6. 06-parent-child

용도 : 정밀도 + 컨텍스트 동시 확보
인덱싱 비용 : 중간 (자식 청크 임베딩)
검색 시점 추가 호출 : 없음
강점 : 단순한 청크 전략으로 큰 효과
약점 : LLM 입력 토큰 증가
구현 난이도 : ★★☆☆☆

## 7. 07-contextual-retrieval

용도 : 청크 단독 모호성 해결
인덱싱 비용 : 매우 큼 (청크당 LLM 호출)
검색 시점 추가 호출 : 없음
강점 : 검색 실패율 평균 49% 감소 (Anthropic)
약점 : 인덱싱 비용 - prompt caching 권장
구현 난이도 : ★★★☆☆

## 8. 08-self-rag

용도 : 검색 불필요 판정 + 청크 필터링
인덱싱 비용 : 낮음
검색 시점 추가 호출 : LLM 1회 (분류) + 청크당 1회 (등급)
강점 : 단순 질문 검색 생략, 무관 청크 제거
약점 : 청크당 LLM 호출 비용 + 분류 정확도 의존
구현 난이도 : ★★★☆☆

## 9. 09-crag

용도 : 검색 신뢰도 평가 + 부족 시 재시도
인덱싱 비용 : 낮음
검색 시점 추가 호출 : LLM 1회 (평가) + 필요 시 1회 (재작성)
강점 : 검색 실패 회복성, 디버깅 쉬움
약점 : 평가자 LLM 의존, 웹 fallback 없으면 코퍼스 외 정보 약함
구현 난이도 : ★★★☆☆

## 10. 10-graphrag (미니멀)

용도 : 전체 조망 / 요약형 질문
인덱싱 비용 : 매우 큼 (청크당 엔티티 추출 LLM 호출)
검색 시점 추가 호출 : 없음
강점 : 도메인 조망 질문에 강함, 엔티티 간 연결 보존
약점 : 인덱싱 비용, 점-단답형 질문에 약함
구현 난이도 : ★★★★☆

## 11. 11-raptor

용도 : 장문/다중 문서 QA, 디테일+요약 동시
인덱싱 비용 : 큼 (클러스터당 LLM 요약)
검색 시점 추가 호출 : 없음
강점 : 글로벌 컨텍스트 보존, 깊이별 매칭 자동화
약점 : 클러스터 수 하이퍼파라미터에 민감
구현 난이도 : ★★★★☆

## 12. 12-agentic-rag (ReAct)

용도 : 다단계/혼합 추론, 도구 사용
인덱싱 비용 : 낮음
검색 시점 추가 호출 : 평균 3-5회 LLM 호출
강점 : 능동적 검색/계산/도구 결합
약점 : 비용/지연 가변, ReAct 파싱 오류 위험
구현 난이도 : ★★★★☆

## 13. 13-adaptive-rag

용도 : 질문 복잡도별 자동 전략 분기
인덱싱 비용 : 낮음
검색 시점 추가 호출 : LLM 1회 (분류) + 분기 전략별 추가
강점 : 비용/품질 자동 최적화, 단순 질문에 빠른 응답
약점 : 분류기 오분류 시 부적절한 전략 선택
구현 난이도 : ★★★☆☆

## 14. 결합 가이드

V1 위주

1. 가장 흔한 운영 구성 - Hybrid + Reranking + Contextual Retrieval
2. 비용 민감 시 - Hybrid + Parent-Child (LLM 호출 0회 추가)
3. 짧고 모호한 질문이 많은 도메인 - Multi-query + Reranking
4. 새로운 도메인 PoC 첫 시도 - Naive → 점수 확보 → Hybrid 추가가 ROI 가장 큼

V2 결합

5. 도메인 조망 질문이 많을 때 - GraphRAG (또는 RAPTOR) + Reranking
6. 검색 비용을 줄여야 할 때 - Adaptive (분기) + 각 분기에 Hybrid/Naive 사용
7. 검색 품질 변동이 큰 도메인 - CRAG + Hybrid Search
8. 도구 사용/혼합 추론이 자주 필요한 어시스턴트 - Agentic RAG + Hybrid 도구
9. 환각 통제가 가장 중요한 답변 - Self-RAG + Reranking + Contextual Retrieval

## 15. 한계

1. 위 강점/약점은 영어/일반 도메인 기준으로 통계된 것이 많습니다. 한국어 + 도메인 특수성에서 동일하게 적용된다는 보장은 없습니다
2. 실제 점수는 본인의 데이터/질문으로 직접 RAGAS 평가를 돌려 확인하는 것을 권장합니다
3. V2 기법은 원논문의 단순화 버전입니다. 풀스펙 GraphRAG, GMM+UMAP RAPTOR, fine-tuned Self-RAG 분류기는 별도 구현 필요
